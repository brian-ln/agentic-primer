# Patterns

Reusable code patterns and solutions discovered through issue-driven development.

## Purpose

This directory contains implementation patterns that can be referenced when solving similar problems. Patterns help ensure consistency across the codebase and reduce time spent on common tasks.

## Pattern Template

When adding a new pattern, use this structure:

```markdown
# Pattern Name

## Problem
What problem does this pattern solve?

## Solution
How does the pattern solve it?

## Example
```code
// Code example
```

## When to Use
- Situation 1
- Situation 2

## When NOT to Use
- Anti-pattern situation

## Related
- Link to related patterns
- Link to decisions
```
