# Patterns

Reusable implementation patterns discovered during issue-driven development.

## What is a Pattern?

A pattern is a proven solution to a recurring problem. Patterns in this directory capture:

- Common implementation approaches
- Code structures that work well
- Integration strategies
- Testing methodologies

## Pattern Template

```markdown
# Pattern Name

## Problem
What problem does this pattern solve?

## Solution
How does this pattern solve it?

## Example
Concrete code or configuration example.

## When to Use
Conditions that make this pattern appropriate.

## When NOT to Use
Conditions that make this pattern inappropriate.

## Related Patterns
Links to similar or complementary patterns.
```

## Adding New Patterns

1. Create a new file: `NNN-pattern-name.md`
2. Use the template above
3. Add an entry to the table in this README
4. Reference the pattern in relevant issues/PRs
