# Insights Catalog

Learned insights from development that help avoid pitfalls and improve solutions.

## Overview

Insights capture lessons learned during development. @copilot references these to avoid known issues and apply proven approaches.

## Insight Template

When adding a new insight, use this template:

```markdown
## Insight Title

**Source:** Issue #NNN or PR #NNN

**Date:** YYYY-MM-DD

**Category:** Bug Fix | Performance | Security | DX | UX

### What We Learned

Brief description of the insight.

### Context

What was the situation that led to this insight?

### Recommendation

What should we do differently in the future?

### Examples

Code or scenario examples if applicable.
```

## How to Add Insights

1. Create a new file: `docs/knowledge/insights/<insight-name>.md`
2. Follow the insight template above
3. Update this index
4. Reference in relevant issues

## Insight Categories

- **Bug Fix** - Lessons from debugging issues
- **Performance** - Optimization discoveries
- **Security** - Security-related learnings
- **DX** - Developer experience improvements
- **UX** - User experience insights
- **Process** - Workflow and process improvements
- **Testing** - Testing strategy learnings
